# 회원가입 및 로그인 플로우 테스트 리포트

## 테스트 실행 결과

- **총 테스트**: 15개
- **통과**: 9개
- **실패**: 6개

## 발견된 문제점

### 1. 셀렉터 문제
- **문제**: `text=회원가입`이 여러 요소와 매칭되어 strict mode violation 발생
- **해결**: `getByRole('heading')` 사용으로 더 정확한 셀렉터 사용

### 2. 이메일 유효성 검사 타이밍 문제
- **문제**: React Hook Form이 blur 시 즉시 검증하지 않고, 폼 제출 시에만 검증
- **해결**: 폼 제출 후 에러 메시지 확인하도록 수정

### 3. 회원가입/로그인 성공 플로우
- **문제**: 실제 Supabase 환경에서 이메일 인증이 필요하거나 다른 문제 발생 가능
- **해결**: 더 긴 대기 시간과 다양한 시나리오 고려

### 4. SignupForm 로그인 링크 문제
- **문제**: 로그인 링크가 "/"로 되어 있어 "/login"으로 수정 필요
- **해결**: ✅ 수정 완료

## 수정 사항

1. ✅ SignupForm의 로그인 링크를 "/login"으로 수정
2. ✅ 테스트 셀렉터를 더 정확하게 수정
3. ✅ 이메일 유효성 검사 테스트를 폼 제출 기반으로 수정
4. ✅ 회원가입/로그인 성공 플로우에 더 긴 대기 시간 추가

## 추가 권장 사항

1. **이메일 인증 설정**: Supabase 대시보드에서 개발 환경에서는 "Auto Confirm" 활성화 고려
2. **테스트 환경 분리**: 실제 Supabase와 테스트용 Supabase 분리 고려
3. **에러 처리 개선**: 더 명확한 에러 메시지와 사용자 피드백
