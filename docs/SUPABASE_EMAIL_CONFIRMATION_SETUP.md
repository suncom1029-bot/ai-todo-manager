# Supabase 이메일 인증 비활성화 설정 가이드

## 개요
개발 환경에서 이메일 인증을 비활성화하여 테스트를 더 쉽게 할 수 있도록 설정하는 방법입니다.

## Supabase 대시보드 설정

### 1. Authentication 설정 접근
1. Supabase 대시보드에 로그인
2. 프로젝트 선택
3. 좌측 메뉴에서 **Authentication** 클릭
4. **Settings** 탭 선택

### 2. 이메일 인증 비활성화
1. **Email Auth** 섹션으로 스크롤
2. **Enable email confirmations** 토글을 **OFF**로 설정
3. 변경사항 저장

### 3. 자동 확인 설정 (선택사항)
개발 환경에서 더 편리하게 테스트하려면:
1. **Email Auth** 섹션에서
2. **Auto Confirm** 옵션을 활성화
   - 이 옵션을 활성화하면 회원가입 시 자동으로 이메일이 확인된 상태로 처리됩니다.

## 코드 레벨 변경사항

코드에서는 이미 이메일 인증을 우회하도록 수정되었습니다:

1. **회원가입 시**: 세션이 없으면 자동으로 로그인 시도
2. **로그인 시**: 개발 환경에서 이메일 인증 오류에 대한 경고 메시지 표시

## 주의사항

⚠️ **중요**: 프로덕션 환경에서는 반드시 이메일 인증을 활성화해야 합니다!

- 개발 환경에서만 이메일 인증을 비활성화하세요
- 프로덕션 환경에서는 보안을 위해 이메일 인증을 필수로 설정하세요

## 테스트

설정 변경 후:
1. 회원가입을 시도하면 즉시 로그인되어야 합니다
2. 이메일 인증 오류 메시지가 표시되지 않아야 합니다
3. 로그인 시 "Email not confirmed" 오류가 발생하지 않아야 합니다

## 문제 해결

### 여전히 이메일 인증 오류가 발생하는 경우:
1. Supabase 대시보드에서 설정이 제대로 저장되었는지 확인
2. 브라우저 캐시를 지우고 다시 시도
3. Supabase 프로젝트를 재시작 (필요한 경우)

### 설정이 적용되지 않는 경우:
1. Supabase 대시보드에서 **Settings** → **API** → **Project URL** 확인
2. 환경 변수 `.env.local`의 `NEXT_PUBLIC_SUPABASE_URL`이 올바른지 확인
3. 개발 서버를 재시작 (`npm run dev`)
